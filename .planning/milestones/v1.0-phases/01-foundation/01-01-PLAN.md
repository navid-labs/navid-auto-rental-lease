    **Step 1: Install Supabase and Prisma dependencies**
    ```bash
    yarn add @supabase/supabase-js @supabase/ssr
    yarn add @prisma/client
    yarn add -D prisma
    ```

    **Step 2: Three Supabase clients**
    Create exactly as documented in RESEARCH.md Patterns 1-2:
    - `src/lib/supabase/client.ts` -- browser client using `createBrowserClient`
    - `src/lib/supabase/server.ts` -- server client using `createServerClient` with cookie handling (try-catch in setAll)
    - `src/lib/supabase/admin.ts` -- admin client using `createClient` with service_role key, no session persistence

    **Step 3: Supabase middleware**
    Create `src/middleware.ts` exactly as documented in RESEARCH.md Pattern 2. Matcher excludes static assets.

    **Step 4: Prisma setup**
    Run `npx prisma init` then replace schema.prisma with the full database schema. Use these models:

    - **Profile**: id (UUID, maps to auth.users), email, name, phone, role (enum: CUSTOMER, DEALER, ADMIN), avatarUrl, createdAt, updatedAt
    - **Brand**: id, name (unique), nameKo, logoUrl, createdAt
    - **CarModel**: id, brandId (FK), name, nameKo, createdAt
    - **Generation**: id, carModelId (FK), name, startYear, endYear, createdAt
    - **Trim**: id, generationId (FK), name, fuelType (enum), engineCC, transmission (enum), createdAt
    - **Vehicle**: id, trimId (FK), dealerId (FK to Profile), year, mileage, color, licensePlate, price, monthlyRental, monthlyLease, status (enum: AVAILABLE, RESERVED, RENTED, LEASED, MAINTENANCE, HIDDEN), description, createdAt, updatedAt
    - **VehicleImage**: id, vehicleId (FK), url, order, isPrimary, createdAt
    - **RentalContract**: id, vehicleId (FK), customerId (FK to Profile), dealerId (FK to Profile), status (enum: DRAFT, PENDING_EKYC, PENDING_APPROVAL, APPROVED, ACTIVE, COMPLETED, CANCELED), startDate, endDate, monthlyPayment, deposit, totalAmount, createdAt, updatedAt
    - **LeaseContract**: id, vehicleId (FK), customerId (FK), dealerId (FK), status (same enum as rental), startDate, endDate, monthlyPayment, deposit, residualValue, residualRate, totalAmount, createdAt, updatedAt
    - **Payment**: id, contractId, contractType (enum: RENTAL, LEASE), amount, paymentDate, status (enum: PENDING, COMPLETED, FAILED, REFUNDED), method, createdAt
    - **Inquiry**: id, customerId (FK to Profile, optional), vehicleId (FK, optional), name, phone, email, message, status (enum: NEW, IN_PROGRESS, COMPLETED), createdAt, updatedAt
    - **ResidualValueRate**: id, brandId (FK), carModelId (FK), year, rate (Decimal), createdAt, updatedAt (unique constraint on brandId+carModelId+year)

    Use `@map` and `@@map` for snake_case table/column names in PostgreSQL. Set datasource to postgresql with `DATABASE_URL` env and `directUrl` for migrations.

    **Step 5: RLS migration**
    Create `prisma/migrations/00000000000000_rls_setup/migration.sql` with:
    - `ALTER TABLE "table_name" ENABLE ROW LEVEL SECURITY;` for every table
    - Default deny-all (no policies = deny for non-superuser)
    - Profile sync trigger: `handle_new_user()` function that inserts Profile on auth.users insert (as documented in RESEARCH.md Pattern 4)

    **Step 6: Prisma client singleton**
    Create `src/lib/db/prisma.ts` with globalThis singleton pattern from RESEARCH.md Pattern 3.

    **Step 7: Shared types**
    Create `src/types/index.ts` with re-exports of Prisma-generated types and any additional app-level types (e.g., `UserRole` type alias).
