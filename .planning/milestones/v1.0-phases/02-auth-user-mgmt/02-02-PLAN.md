  <name>Task 1: Profile update and admin role change Server Actions with my page</name>
  <files>

  <behavior>
    - updateProfile rejects if user not authenticated (returns error)
    - updateProfile validates input with profileUpdateSchema
    - updateProfile updates name and phone via Prisma for the current user only
    - updateProfile does NOT allow changing role, email, or id
    - updateProfile returns success after update
    - changeUserRole rejects if caller is not ADMIN role
    - changeUserRole updates target user's role in profiles table AND auth.users app_metadata
    - changeUserRole revalidates /admin/users path
  </behavior>
  <action>
    1. Create `src/features/auth/actions/profile.ts` ('use server'):
       - `updateProfile(formData: FormData)`:
         - Get current user via `getCurrentUser()`. If null, return `{ error: '로그인이 필요합니다.' }`
         - Parse FormData with `profileUpdateSchema`. On error, return `{ error: '입력 정보를 확인해주세요.' }`
         - Update via `prisma.profile.update({ where: { id: user.id }, data: { name, phone } })`
         - Call `revalidatePath('/mypage')`
         - Return `{ success: true }`
       - `changeUserRole(userId: string, newRole: UserRole)`:
         - Get current user. If null or role !== 'ADMIN', return `{ error: '권한이 없습니다.' }`
         - Validate newRole is one of CUSTOMER/DEALER/ADMIN
         - Update profiles via `prisma.profile.update({ where: { id: userId }, data: { role: newRole } })`
         - Update auth.users via `createAdminClient().auth.admin.updateUserById(userId, { app_metadata: { role: newRole } })`
         - Call `revalidatePath('/admin/users')`
         - Return `{ success: true }`

    2. Create `src/features/auth/actions/profile.test.ts`:
       - Test updateProfile: unauthorized returns error, invalid input returns error, valid input updates profile
       - Test changeUserRole: non-admin returns error, admin can change roles, revalidates path
       - Mock getCurrentUser, prisma.profile.update, createAdminClient

    3. Create `src/features/auth/components/profile-form.tsx` ('use client'):
       - Use react-hook-form with zodResolver(profileUpdateSchema)
       - Fields: name (Input, required), phone (Input, optional)
       - Show email as read-only display (not editable)
       - Show current role as badge (not editable by user)
       - useTransition for pending state
       - On success, show toast or success message
       - Korean labels: 이름, 전화번호, 이메일, 역할

    4. Create `src/app/(protected)/layout.tsx`:
       - Simple pass-through layout (middleware handles protection, no duplicate auth check needed)
       - Use the existing public layout structure (header + footer)

    5. Create `src/app/(protected)/mypage/page.tsx`:
       - Server Component
       - Fetch current user via `getCurrentUser()`
       - If no user (shouldn't happen due to middleware), redirect to /login
       - Display user info card + ProfileForm
       - Korean title: 마이페이지
       - Set metadata: `{ title: '마이페이지 | 나비드오토' }`
  </action>
