<objective>
Build the pure calculation layer and data access layer for Phase 6.

Purpose: Establish tested business logic (rental/lease calculations, monthly estimate, residual rate CRUD) that the UI plans will consume. Output: Tested calculation functions in `src/lib/finance/calculate.ts`, Zod schemas in `src/features/pricing/schemas/`, and Server Actions for residual rate CRUD in `src/features/pricing/actions/`.

  <behavior>
    ## Calculation Functions (src/lib/finance/calculate.ts)

    ### calculateRental(vehiclePrice, periodMonths, deposit)
    - Test: calculateRental(36_000_000, 36, 0) => monthlyPayment: 1_000_000, totalCost: 36_000_000
    - Test: calculateRental(36_000_000, 36, 3_600_000) => monthlyPayment: 900_000 (deduct deposit from principal)
    - Test: calculateRental(50_000_000, 60, 10_000_000) => monthlyPayment: ~666_667
    - Test: returns { monthlyPayment, deposit, totalCost, periodMonths }

    ### calculateLease(vehiclePrice, periodMonths, deposit, residualRate)
    - Uses existing pmt() function for proper financial calculation (not simple division)
    - Test: calculateLease(36_000_000, 36, 0, 0.45) => monthlyPayment less than rental equivalent (residual excluded from financing)
    - Test: with residualRate=0, lease monthly equals rental monthly (no residual benefit)
    - Test: returns { monthlyPayment, deposit, residualValue, residualRate, totalCost, periodMonths }
    - Residual value = vehiclePrice * residualRate

    ### estimateMonthlyRental(vehiclePrice, defaultPeriod=36)
    - Test: estimateMonthlyRental(36_000_000) => 1_000_000
    - Test: estimateMonthlyRental(25_000_000, 48) => ~520_833
    - Test: estimateMonthlyRental(0) => 0 (guard zero price)

    ### formatEstimate(monthlyAmount)
    - Test: formatEstimate(320_000) => "렌탈 월 ~32만원부터"
    - Test: formatEstimate(1_000_000) => "렌탈 월 ~100만원부터"
    - Test: formatEstimate(0) => returns empty or dash (no estimate for zero)

    ### Edge cases
    - All functions handle zero deposit gracefully
    - All period options valid: 12, 24, 36, 48, 60
    - Guard against zero/negative vehicle price

    ## Zod Schemas (src/features/pricing/schemas/residual-value.ts)
    - residualValueRateSchema: { brandId: uuid, carModelId: uuid, year: number (2010-2030), rate: number (0.01-0.99) }

    ## Server Actions (src/features/pricing/actions/residual-rate.ts)
    - getResidualRate(brandId, carModelId, year): returns rate as number | null
    - getResidualRates(brandId?): returns array with brand/model names for admin table
    - upsertResidualRate(data): admin-only, validates with Zod, upserts, revalidates
    - deleteResidualRate(id): admin-only, deletes, revalidates
  </behavior>
  <implementation>
    1. Create `src/lib/finance/calculate.ts` with four exported functions:
       - `calculateRental`: Simple division formula (vehiclePrice - deposit) / periodMonths. Returns RentalResult type.
       - `calculateLease`: Use existing `pmt()` from `./pmt` for proper lease calculation. residualValue = vehiclePrice * residualRate. Use `pmt(0.084/12, periodMonths, vehiclePrice - deposit, -residualValue)` with domestic default rate 0.084. Returns LeaseResult type.
       - `estimateMonthlyRental`: Quick estimate, `Math.round(vehiclePrice / defaultPeriod)`. Guard price <= 0 returns 0.
       - `formatEstimate`: Convert to 만원 units, return "렌탈 월 ~{N}만원부터". Return empty string for zero.

    2. Update `src/lib/finance/index.ts` to re-export new functions and types.

    3. Create `src/features/pricing/schemas/residual-value.ts` with Zod schema for admin form validation.

    4. Create `src/features/pricing/actions/residual-rate.ts` with Server Actions following project pattern:
       - `'use server'` directive
       - getCurrentUser() + role check for mutations
       - Prisma queries with includes for brand/model names
       - `revalidatePath('/admin/residual-value')` after mutations
       - Convert Prisma Decimal to number with `.toNumber()` (CRITICAL: avoid [object Object])

    IMPORTANT: For `calculateLease`, use a simplified annual rate constant (e.g., 0.084 for domestic) rather than requiring all the inputs of the full `calculateQuote`. The full calculator is for internal/contract use; this simplified version is for the public comparison UI.
