## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | **Phase 18 primary value** -- layout `<main>` top padding (`pt-6`), admin content padding |
| xl | 32px | Layout gaps (upper bound for content top margin per GLBL-02) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **52px** -- Navigation mega menu bar height (GLBL-01). Not a standard scale value. Expressed as `h-[52px]` arbitrary value in Tailwind.
- **68px** -- Main header bar height. Pre-existing, NOT modified in this phase.

### Phase 18 Spacing Contract

These are the exact spacing values this phase implements:

| Element | Current Value | Target Value | Tailwind Class | Requirement |
|---------|--------------|-------------|----------------|-------------|
| Mega menu nav bar height | 48px (`h-12`) | 52px | `h-[52px]` | GLBL-01 |
| Public layout `<main>` top padding | 0px (none) | 24px | `pt-6` | GLBL-02 |
| Protected layout `<main>` top padding | 0px (none) | 24px | `pt-6` | GLBL-02 |
| Admin layout `<main>` padding | 24px (`p-6`) | 24px (`p-6`) -- no change | `p-6` (keep) | GLBL-03 |
| Dealer layout `<main>` padding | 24px (`p-6`) | 24px (`p-6`) -- no change | `p-6` (keep) | GLBL-03 |

### Edge-to-Edge Override Pattern

Pages with intentional edge-to-edge content (hero banner, vehicle gallery) must counteract the layout-level `pt-6` with a negative margin:

| Page | Component | Override Class | Reason |
|------|-----------|---------------|--------|
| `/` (homepage) | HeroBanner wrapper | `-mt-6` | Hero must touch nav bottom edge |
| `/vehicles/[id]` | Gallery wrapper | `-mt-6` | Gallery must touch nav bottom edge |

### Per-Page Padding De-duplication

When layout-level `pt-6` is added, existing per-page top padding MUST be removed to prevent double-padding:

| Page | Current Class | Target Change | Reason |
|------|--------------|--------------|--------|
| `/vehicles` | `pt-4` on breadcrumb wrapper | Remove `pt-4` | Layout provides 24px |
| `/sell` | `pt-6` on breadcrumb wrapper | Remove `pt-6` | Layout provides 24px |
| `/calculator` | `py-10` on wrapper | Change to `pb-10` | Layout provides top; keep bottom |
| `/inquiry` | `py-12` on wrapper | Change to `pb-12` | Layout provides top; keep bottom |

## Visual Validation Checklist

Since this phase has no copywriting or new components, the primary validation is visual spacing correctness across all page types:

| Check | Expected Result | Pages to Verify |
|-------|----------------|-----------------|
| Mega menu nav height | 52px tall, category links vertically centered | All public pages (desktop only -- mobile has no mega menu) |
| Content top breathing room | 24px gap between sticky header bottom and first content element | `/vehicles`, `/sell`, `/calculator`, `/inquiry`, all protected pages |
| Edge-to-edge hero | Hero banner touches nav bottom edge with NO gap | `/` homepage only |
| Edge-to-edge gallery | Vehicle gallery touches nav bottom edge with NO gap | `/vehicles/[id]` only |
| No double padding | No page shows 48px+ gap (would indicate duplicate padding) | All public pages |
| Admin content padding | 24px padding on all sides of main content area, unchanged from current | `/admin/*` pages |
| Scroll-spy accuracy | Correct section tab highlighted while scrolling vehicle detail page | `/vehicles/[id]` (rootMargin offset may need +4px adjustment) |

## Anti-Patterns

These patterns MUST NOT appear in the implementation:

1. **Double padding** -- Adding `pt-6` to layout AND keeping per-page `pt-*`/`py-*` values. Each page's existing top padding must be audited and removed.
2. **Margin-top on `<main>`** -- Using `mt-6` instead of `pt-6`. Margin creates a visible gap between the sticky header and content during scroll. Padding does not.
3. **Changing main header bar height** -- GLBL-01 targets the mega menu nav bar only (`h-12` to `h-[52px]`). The main header bar (`h-[68px]`) must NOT change.
4. **CSS custom properties for this scope** -- The codebase uses Tailwind classes exclusively. Do not introduce `--nav-height` or `--content-top-gap` CSS variables. Tailwind `h-[52px]` and `pt-6` are sufficient.
5. **Modifying admin sidebar structure** -- GLBL-03 means "same content breathing room," NOT "same navigation layout." Admin `p-6` is already compliant. No structural changes needed.
