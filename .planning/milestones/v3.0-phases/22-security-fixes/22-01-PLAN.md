    **2. Auth guard for ekyc send-code endpoint:**
    In `src/app/api/contracts/ekyc/send-code/route.ts`:
    - Add `import { requireAuth } from '@/lib/api/auth'` at the top
    - At the start of the POST handler, add:
      ```typescript
      const auth = await requireAuth()
      if (auth.error) return auth.error
      ```
    - Use `requireAuth()` (not `requireAdmin()`) because any authenticated user can request eKYC verification codes, per CONTEXT.md decision.

    **3. Password hashing migration:**
    In `src/features/settings/mutations/auth.ts`:
    - Remove the `const DEFAULT_PASSWORD = 'admin1234'` line
    - Replace with `const DEFAULT_PASSWORD_HASH` that is the argon2id hash of 'admin1234'. Generate at module level using a pre-computed hash string (do NOT call Bun.password.hash at import time -- use a constant).
    - To get the hash value, run `await Bun.password.hash('admin1234', { algorithm: 'argon2id' })` once during development and paste the result.
    - Actually, since argon2 hashes include a random salt, we cannot pre-compute a static hash. Instead, use this approach:
      - Keep `const DEFAULT_PASSWORD = 'admin1234'` as a fallback
      - In `verifySettingsPasswordMutation`, after fetching the record:
        - If `record` exists and `record.value` starts with `$argon2`: use `await Bun.password.verify(password, record.value)` to compare
        - If `record` exists and `record.value` does NOT start with `$argon2`: compare as plaintext (`password === record.value`) for backwards compatibility
        - If `record` is null: compare against `DEFAULT_PASSWORD` as plaintext (`password === DEFAULT_PASSWORD`)
      - Return `{ error: '...' }` if verification fails, `{ success: true }` if it passes

    **4. Tests:**
    Create `tests/unit/features/settings/verify-password.test.ts`:
    - Mock `@/lib/db/prisma` to control `prisma.defaultSetting.findUnique` return values
    - Test empty password returns error
    - Test wrong password against argon2 hash returns error
    - Test correct password against argon2 hash returns success (use `await Bun.password.hash('testpass', { algorithm: 'argon2id' })` in test setup)
    - Test plaintext fallback: stored value is 'admin1234' (no $argon2 prefix), password is 'admin1234' -> success
    - Test no record in DB: password 'admin1234' matches DEFAULT_PASSWORD -> success

    **Note:** The `/api/inquiry` and `/api/vehicles/[id]/inquiry` endpoints intentionally remain public (contact forms), per CONTEXT.md decision. Rate limiting deferred to v4.0.
