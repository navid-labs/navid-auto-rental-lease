  <action>
    1. Create `src/features/contracts/actions/approve-contract.ts`:
       - 'use server' directive
       - approveContract(contractId: string, contractType: 'RENTAL' | 'LEASE', action: 'APPROVED' | 'CANCELED', reason?: string)
       - Authenticate with getCurrentUser(), require ADMIN role
       - Fetch contract, validate current status allows transition via canTransitionContract()
       - Use prisma.$transaction:
         a. Update contract status to action value
         b. If APPROVED: update vehicle status from RESERVED to RENTED (for RENTAL) or LEASED (for LEASE), set contract startDate to now(), calculate endDate from periodMonths
         c. If CANCELED: update vehicle status back to AVAILABLE
       - Return { success: true } or { error: string }

       - Test invalid transition rejection
       - Use vi.hoisted() for mocks

    3. Create `src/features/contracts/hooks/use-contract-realtime.ts`:
       - 'use client' hook
       - useContractRealtime(contractId: string, contractType: 'RENTAL' | 'LEASE', onUpdate: (newStatus: ContractStatus) => void)
       - Determines table name: contractType === 'RENTAL' ? 'rental_contracts' : 'lease_contracts'
       - Creates Supabase channel: `contract-${contractId}`
       - Subscribes to postgres_changes event 'UPDATE' on the appropriate table with filter `id=eq.${contractId}`
       - Extracts new status from payload.new.status and calls onUpdate
       - Cleanup: supabase.removeChannel(channel) in useEffect return
       - Add fallback: if channel errors, log warning (no polling fallback for v1, per research recommendation)

    4. Create `src/features/contracts/components/admin-contract-list.tsx`:
       - 'use client' component (needs interactive approval actions)

       - Each row shows vehicle name (brand + model + year), customer name, contract type badge, monthly payment (formatKRW), ContractStatusBadge, createdAt (formatDate), action buttons
       - For PENDING_APPROVAL status: "승인" (green) and "반려" (red) buttons
       - "반려" opens a Dialog with reason textarea (Korean preset reasons as quick-select chips: "서류 미비", "신용 확인 필요", "차량 상태 변경")
