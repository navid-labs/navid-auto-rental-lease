1. Add two new models to prisma/schema.prisma:

```prisma
model PromoRate {
  id        String   @id @default(uuid()) @db.Uuid
  brandId   String   @map("brand_id") @db.Uuid
  rate      Decimal  // 0.03 = 3%
  label     String?
  isActive  Boolean  @default(true) @map("is_active")
  createdAt DateTime @default(now()) @map("created_at")
  updatedAt DateTime @updatedAt @map("updated_at")
  brand Brand @relation(fields: [brandId], references: [id])
  @@unique([brandId])
  @@map("promo_rates")
}

model DefaultSetting {
  id        String   @id @default(uuid()) @db.Uuid
  key       String   @unique
  value     String
  label     String
  updatedAt DateTime @updatedAt @map("updated_at")
  @@map("default_settings")
}
```

Add `promoRates PromoRate[]` to the Brand model relations.

Note: Subsidy data is already stored per InventoryItem row (subsidy column). DefaultSetting will store subsidy-related defaults as key-value pairs (e.g., key: "default_subsidy", value: "500000").

Run `yarn db:push` and `yarn db:generate` after schema changes.

2. Create Zod schemas in settings.ts:
- promoRateSchema: { brandId: z.string().uuid(), rate: z.number().min(0).max(1), label: z.string().optional() }
- defaultSettingSchema: { key: z.string().min(1), value: z.string().min(1), label: z.string().min(1) }
- settingsPasswordSchema: { password: z.string().min(1) }

3. Create settings-auth.ts server action:
- verifySettingsPassword(password: string): checks against DefaultSetting where key = "settings_password". If no record exists, default password is "admin1234". Returns { success: true } or { error: string }.

4. Create settings.ts server actions (all with admin role check):
- getPromoRates(): findMany with brand include, ordered by brand name
- upsertPromoRate(data): Zod validate, upsert by brandId, revalidatePath('/admin/settings')
- deletePromoRate(id): delete by id, revalidatePath
- getDefaultSettings(): findMany ordered by key
- upsertDefaultSetting(data): Zod validate, upsert by key, revalidatePath
- deleteDefaultSetting(id): delete by id, revalidatePath

Use ActionResult pattern: `{ success: true } | { error: string }`.
Use Zod 4 .issues (not .errors) for validation error access.
