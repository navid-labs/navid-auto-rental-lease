1. Create inventory-upload.ts Zod schema:
- csvRowSchema: validates a single parsed row matching InventoryTableRow fields.
  - category: z.enum(['STRATEGIC', 'GENERAL'])
  - itemNumber: z.string().min(1)
  - representModel: z.string().min(1)
  - modelName: z.string().min(1)
  - modelYear: z.coerce.number().int().min(2000)
  - exteriorColor: z.string().min(1)
  - interiorColor: z.string().min(1)
  - price: z.coerce.number().int().min(0)
  - subsidy: z.coerce.number().int().default(0)
  - availableQuantity: z.coerce.number().int().min(0)
  - immediateQuantity: z.coerce.number().int().min(0)
  - Optional fields: promotion, options, productionDate, notice (all z.string().optional())

2. Create inventory-upload.ts server action:
- uploadInventoryCsv(formData: FormData): Promise<{ success: true; count: number } | { error: string; rowErrors?: { row: number; message: string }[] }>
  - Admin role check
  - Get file from FormData, read as text
  - Handle BOM: text.replace(/^\uFEFF/, '')
  - Handle CRLF: text.replace(/\r\n/g, '\n')
  - Split into lines, first line = headers
  - Korean header mapping (same as json-adapter): 구분 -> category (use parseCategory logic), 번호 -> itemNumber, etc.
  - For each data row: map headers to fields, validate with csvRowSchema, collect errors with row numbers
  - If errors exist, return { error: "N행에서 오류 발생", rowErrors }
  - If valid: deleteMany + createMany (same pattern as load-inventory.ts)
  - After successful upload, upsert a "last_inventory_upload" key in a simple approach: store timestamp in the InventoryItem loadedAt field (already exists).
  - revalidatePath('/admin/inventory')
  - Return { success: true, count }

- getLastUploadTime(): Promise<Date | null>
  - Query the most recent loadedAt from InventoryItem table
  - Return null if no items exist

Strip commas from numeric fields before parsing (price fields may have "25,000,000" format). Reuse parseNumber logic from json-adapter.

3. Create csv-upload-form.tsx (client component):
- File input accepting .csv files
- "업로드" button, disabled while uploading (useTransition)
- On submit: create FormData, call uploadInventoryCsv
- Show result: success message with count, or error with row-level details
- Style with existing Card/Button components

4. Create upload-status.tsx (client component):
- Receives lastUploadTime: Date | null prop
- Displays "마지막 업데이트: YYYY-MM-DD HH:mm" using formatDate utility or toLocaleString('ko-KR')
- If null, show "업로드된 데이터 없음"

5. Integrate into existing inventory page (src/app/admin/inventory/page.tsx):
- Add CsvUploadForm and UploadStatus components above the existing inventory table
- Fetch lastUploadTime from getLastUploadTime() in the server component
- Keep existing "데이터 조회" button and table functionality intact
