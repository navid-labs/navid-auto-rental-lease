# Phase 15 — UI Design Contract: Search & Listing Page

> Visual and interaction contract for the K Car-style search/listing page redesign.
> Generated by gsd-ui-researcher. All values are final — no ad-hoc styling decisions during execution.

### SEARCH-01 Note
REQUIREMENTS.md lists 14 filters including "판매구분". This spec expands to **15 filters** by adding "월 납입금" (monthly payment slider) per CONTEXT.md differentiator decision while retaining "판매구분" as filter #14. Total: 15 filter sections.

### Token Prerequisites
Executor must add `--badge-discount: hsl(24 95% 53%)` to `globals.css` and map it via `--color-badge-discount` in `@theme inline` before using `bg-badge-discount`.

## Dark Mode Considerations

All colors use CSS variables that have `.dark` overrides in `globals.css`. No hardcoded hex values in component code.

Key dark mode changes (already defined in globals.css):
- `--background`: hsl(220 40% 8%)
- `--card`: hsl(220 35% 12%)
- `--border`: hsl(220 20% 20%)
- `--card-hover`: hsl(220 25% 15%)
- `--text-price`: hsl(217 91% 70%)
- `--badge-*`: Slightly brighter versions for dark backgrounds

**Implementation rule:** Use only Tailwind semantic classes (bg-card, text-foreground, border-border, etc.). Never hardcode hex/hsl values except for color chip swatches in the color filter.
