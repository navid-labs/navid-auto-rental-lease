## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, vehicle card info padding (target: `p-4`) |
| lg | 24px | Section padding, grid gaps (target: `gap-6`), search-to-breadcrumb transition |
| xl | 32px | Quick links section padding (`py-8`), partner logos section padding |
| 2xl | 40px | Section-level vertical padding (`py-10`) for mid-weight sections |
| 3xl | 48px | Section-level vertical padding (`py-12`) for heavyweight sections (md breakpoint) |
| 4xl | 80px | **Phase 19 primary value** -- inter-section visible gap target (`space-y-20` equivalent) |

Exceptions:
- **52px** -- Navigation mega menu bar height (Phase 18, unchanged). `h-[52px]`
- **80px+** -- Target inter-section visible gap. Achieved via the SUM of adjacent section paddings, NOT via a single margin value. Expressed as `py-10` / `py-12` combinations.

### Anti-Patterns

These patterns MUST NOT appear in the implementation:

1. **Do NOT use `space-y-*` on homepage sections.** Sections have different background colors (white, #F9FAFB). `space-y-*` adds margin-top which creates visible page-background strips between colored sections. Use per-section padding increases instead.
2. **Do NOT change the `-mt-6` on HeroBanner wrapper.** This is the Phase 18 edge-to-edge override. Add spacing below the hero via `mt-10` wrapper on HeroSearchBox.
3. **Do NOT change vehicle card image aspect ratio or container.** SRCH-03 targets only the text info area (`p-3.5` to `p-4`), not the image.
4. **Do NOT change the search page sidebar width (280px) or sidebar gap (`gap-8`).** Only vehicle grid gap and card internal padding are in scope.
5. **Do NOT add both `mb-4` and `my-4` to QuickFilterBadges.** Replace `mb-4` with `my-4` (Tailwind v4 cascade: last class wins, but avoid ambiguity by removing the original).
