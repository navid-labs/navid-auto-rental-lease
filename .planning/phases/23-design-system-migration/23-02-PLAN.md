## Style Object Replacements (for inline style={{ }})

When hex values appear in `style={{ color: '#0D0D0D' }}` objects:
- Replace with CSS variable reference: `style={{ color: 'var(--foreground)' }}`
- Or better: convert to Tailwind class and remove the style prop entirely
- Example: `style={{ color: '#999999' }}` on an `<option>` element can become `className="text-text-tertiary"`
- Note: `<option>` elements have limited CSS support -- inline styles are acceptable here

## Gradient/Overlay Exceptions (DO NOT REPLACE)

- `linear-gradient(160deg, #0a0f1e 0%, #0d1428 30%, #111827 60%, #0a0d18 100%)` in hero-section.tsx -- keep as-is (decorative gradient)
- `#000000AA` overlay in hero-section.tsx -- keep as-is (opacity overlay)
- Dark navy gradient stops (`#0f1e3c`, `#1a3a6e`, `#152849`, etc.) -- keep as-is (decorative)
