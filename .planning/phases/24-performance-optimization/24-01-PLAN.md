    Modify `src/app/(public)/layout.tsx` to dynamically import the 3 framer-motion layout components instead of static imports.

    1. Add `'use client'` is NOT needed -- the layout is a Server Component. `next/dynamic` works in Server Components.

    2. Add import at top:
       ```typescript
       import dynamic from 'next/dynamic'
       ```

    3. Replace the 3 static imports:
       ```typescript
       // REMOVE these:
       import { FloatingCTA } from '@/components/layout/floating-cta'
       import { RecentlyViewedDrawer } from '@/components/layout/recently-viewed-drawer'
       import { ComparisonBar } from '@/components/layout/comparison-bar'

       // REPLACE with:
       const FloatingCTA = dynamic(
         () => import('@/components/layout/floating-cta').then(mod => mod.FloatingCTA),
         { ssr: false }
       )
       const RecentlyViewedDrawer = dynamic(
         () => import('@/components/layout/recently-viewed-drawer').then(mod => mod.RecentlyViewedDrawer),
         { ssr: false }
       )
       const ComparisonBar = dynamic(
         () => import('@/components/layout/comparison-bar').then(mod => mod.ComparisonBar),
         { ssr: false }
       )
       ```

    4. The JSX usage of these components stays EXACTLY the same -- no changes to the template.

    5. `ssr: false` is correct because:
       - FloatingCTA: Uses window.scrollY, only visible after scroll
       - RecentlyViewedDrawer: Uses Zustand store hydration, conditionally renders
       - ComparisonBar: Uses Zustand store, conditionally renders
       - All 3 already have `'use client'` directives and mount checks

    NOTE: Do NOT add loading fallbacks. These components render nothing by default (FloatingCTA shows after scroll, others show after user interaction). A loading skeleton would flash unnecessarily.
