## 발견 이슈

### 🔴 P0 — Critical

#### ISSUE-001: 어드민 페이지 인증 미적용
- **경로**: `/admin`, `/admin/leads`, `/admin/listings`, `/admin/escrow`
- **증상**: 비로그인 상태에서도 접근 가능
- **영향**: 관리자 권한 우회 가능 (보안 사고 위험)
- **재현**: 로그아웃 상태로 `/admin` 직접 접속
- **기대**: `/login` 리다이렉트 또는 403
- **수정 포인트**:
  - `src/app/admin/layout.tsx`에 `requireRole("ADMIN")` 게이트 적용
  - middleware.ts에서 `/admin/*` 경로 서버단 인증 검증

### 🟠 P1 — High

#### ISSUE-002: /list 페이지 콘솔 에러 다수
- **경로**: `/list`
- **증상**:
  - `401` 1건 (인증 관련)
  - `400 Bad Request` 6건 (API 파라미터 오류 추정)
  - CSP 경고
- **영향**: 잘못된 요청 누적 → 백엔드 부하 + 사용자 UX 저해
- **수정 포인트**:
  - `/api/listings` 호출 파라미터 검증
  - `/api/favorites/my` — 비로그인 시 401 방지 (conditional fetch)
  - `next.config.*`에 CSP 헤더 정비

### 🟡 P2 — Medium

#### ISSUE-003~006: 푸터 링크 404 (페이지 미존재)
- **경로**: `/used`, `/terms`, `/privacy`, `/faq`
- **증상**: 푸터에서 링크는 노출되지만 페이지 파일 없음 → 404
- **영향**: 법적 고지(개인정보·이용약관) 부재 → 서비스 출시 블로커
- **수정 포인트**:
  - `src/app/(public)/used/page.tsx` (중고 리스/렌트 인덱스)
  - `src/app/(public)/terms/page.tsx` — 이용약관
  - `src/app/(public)/privacy/page.tsx` — 개인정보처리방침
  - `src/app/(public)/faq/page.tsx` — 자주 묻는 질문
  - 또는 푸터에서 해당 링크 임시 제거

#### ISSUE-007: 상세 페이지 이미지 갤러리 비어있음
- **경로**: `/detail/[id]`
- **증상**: 갤러리 영역이 렌더되지만 이미지(placeholder 포함) 미로드
- **영향**: 매물 신뢰도 저하 (핵심 구매 결정 요소)
- **수정 포인트**:
  - seed 데이터에 `ListingImage` 레코드 추가
  - 이미지 없을 때 fallback UI (`features/listings/gallery`)
  - Supabase Storage URL 유효성 검증
